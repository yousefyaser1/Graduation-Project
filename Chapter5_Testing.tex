\chapter{Testing and Results}

This chapter presents a systematic evaluation of the proposed skin disease classification system. Testing is conducted exclusively on the held-out validation split to preserve the integrity of the test set for any subsequent fine-tuning. Evaluation covers six dimensions: CNN classifier performance across all training phases, VAE anomaly detection reliability, Score-CAM explainability output quality, and end-to-end mobile application behaviour — including unit testing of individual pipeline components, boundary and edge case robustness, usability evaluation with real users, real-world use case scenario walkthroughs, and on-device performance measurement.

\subsection{Evaluation Metrics}

Performance is quantified using the following standard metrics:

\begin{itemize}
    \item \textbf{Overall accuracy} — fraction of correctly classified validation samples.
    \item \textbf{Per-class precision} — fraction of predicted positives that are truly positive.
    \item \textbf{Per-class recall (sensitivity)} — fraction of true positives correctly retrieved.
    \item \textbf{Per-class F1-score} — harmonic mean of precision and recall.
    \item \textbf{Confusion matrix} — absolute count of correct and misclassified predictions per class pair.
    \item \textbf{ROC curve and AUC} — separability of each class across all decision thresholds.
\end{itemize}

\section{VAE Anomaly Detection Evaluation}
% ============================================================

The Variational Autoencoder serves as the first stage of the pipeline, gating all inputs before they reach the CNN. It was trained exclusively on Normal skin images and, at inference time, applies a patch-based sliding window to each input image to compute a local reconstruction error.

\subsection{Inference Mechanism}

Rather than computing a single image-level MSE, the VAE operates on overlapping 64$\times$64 pixel patches extracted with a stride of 32 pixels from each resized 224$\times$224 input. Each patch is independently encoded and decoded; its per-patch MSE reconstruction error is compared against a patch-level threshold $\tau_p = 0.008$. The fraction of patches whose error exceeds $\tau_p$ is termed the \textit{anomaly ratio}. If the anomaly ratio exceeds a second threshold $\tau_r = 0.20$, the image is classified as anomalous and forwarded to the CNN; otherwise it is returned as ``Normal'' without further processing.

    \label{fig:vae_histogram}
\end{figure}

\noindent The VAE achieves an overall TPR of 68.92\% at the deployed threshold $\tau_r = 0.20$, meaning 224 of the 325 diseased validation images are correctly flagged as anomalous and forwarded to the CNN classifier. The remaining 31.08\% are classified as ``Normal'' at the gating stage, producing system-level false negatives.

The most significant gap appears in the Acne class (TPR 55.91\%), which is attributable to the focal, small-area nature of acne lesions. Because acne papules and pustules occupy only a fraction of the skin surface, the majority of 64$\times$64 patches extracted from an Acne image may depict largely normal perilesional skin, keeping the overall anomaly ratio below the decision threshold. Eczema and Tinea, which present with more diffuse erythema and scaling that covers a larger patch area, yield higher TPRs of 76.76\% and 70.00\% respectively.

This VAE sensitivity represents a known limitation of the current system. The threshold $\tau_r$ is a tunable parameter; a sensitivity analysis shows that reducing it to 0.10 raises the overall TPR to 79.08\%, though this would also increase the false positive rate (Normal images wrongly forwarded to the CNN) — a trade-off that requires Normal-class evaluation data to characterise fully. Threshold optimisation guided by clinician input and a balanced Normal/diseased evaluation set is identified as a priority for future work.

\section{Score-CAM Explainability Evaluation}

\subsection{Usability Testing}

Usability testing was conducted to assess whether users of varying technical backgrounds could operate the application independently and arrive at a result without confusion or instruction.

Three participants were recruited, each representing a distinct user archetype:

\begin{itemize}
    \item \textbf{Participant A} — a patient with no medical or technical background.

% ── Scenario 1 ────────────────────────────────────────────────────────────────
\subsubsection{Scenario 1: Patient Self-Check — Tinea}

\textbf{Actor:} General User (Patient). \\
\textbf{Precondition:} The app is installed and the patient is logged in. \\
\textbf{Trigger:} The patient notices a red, circular rash on their right arm and suspects a fungal infection.

\paragraph{Main Success Scenario.}
The patient taps \textit{``Tap To Scan''} and captures the affected area.

\textbf{Stage 1 — VAE:} The image is passed to the VAE model. The model fails to reconstruct the lesion area, as it was trained only on normal skin. The reconstruction error exceeds the anomaly threshold ($\tau_r = 0.20$) and the image is classified as \textbf{Anomalous}.

\textbf{Stage 2 — CNN:} The EfficientNetB2+B3 ensemble analyses the image and identifies characteristic features: circular shape and scaly border. It returns the top prediction: \textbf{``Tinea'' at 94\% confidence}.

\textbf{Stage 3 — Score-CAM:} The heatmap highlights the annular, scaly border of the lesion — the clinically defining feature of Tinea.

\textbf{Result:} The Result screen displays the diagnosis, confidence score, probability bars, and the Score-CAM heatmap overlay. The patient saves the result to \textit{Scan History} to show to a dermatologist.

\section{Chapter Summary}
% ============================================================

This chapter reported the evaluation of all three pipeline stages against the held-out validation set and on a physical Android device, supplemented by unit, boundary, usability, and scenario-based testing of the mobile application.

The B2+B3 ensemble with 20-step TTA achieved \textbf{92.62\%} overall accuracy on 325 validation samples, with per-class F1-scores of 0.9263 (Acne), 0.9291 (Eczema), and 0.9213 (Tinea), and AUC values exceeding 0.97 for all three classes. The dominant misclassification is Eczema confused with Acne, which is clinically expected given their shared visual presentation. The VAE anomaly detector achieved an overall TPR of 68.92\%; Acne's lower TPR (55.91\%) reflects the focal, small-area nature of acne lesions, and threshold optimisation is identified as future work. Score-CAM heatmaps were visually consistent with clinically relevant lesion regions across all three disease classes.

Unit tests confirmed that all individual AI pipeline components and data layer operations produce correct outputs in isolation, including round-trip serialisation fidelity for scan records. Boundary and edge case testing confirmed the application handles extreme image resolutions, corrupted files, screen rotation, and rapid repeated usage without crashes; the only unresolved limitation is the absence of an input validation filter for non-skin images. Usability testing with three participants across distinct user archetypes confirmed that all core flows are completable without instruction, and one discovered discoverability issue (the notes input field) was corrected. Five use case scenarios demonstrated correct end-to-end system behaviour across all three disease classes, the normal-skin early-exit path, and the history review workflow. On-device inference satisfies the $\leq$\,2-second latency target and the application passed all twelve functional test cases including offline operation.
